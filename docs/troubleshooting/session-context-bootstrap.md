# 세션 시작 시 본인 역할 맥락 강제 로딩

## 배경
- 각 워크플로우 페이즈·회의·태스크 실행은 **새 Claude Code 세션**으로 돌아간다.
- 세션 간 대화·메모리는 **공유되지 않는다** — 파일시스템(docs/, kanban.json, sprints.json, metrics.json)과 메시지 큐가 유일한 장기 기억.
- 따라서 세션이 시작될 때 에이전트가 본인 역할과 관련된 맥락을 스스로 읽어오지 않으면, 이전 결정·회의 합의·진행 상황을 놓친 채 일관성 없는 산출물을 만든다.

## 증상 예시
- CTO 가 아키텍처 결정을 내릴 때, 지난 주에 이미 docs/decisions/ 에 합의된 내용과 모순되는 결정을 내림.
- PO 가 기획 문서를 새로 작성하면서 기존 기획의 메트릭 계획을 놓침.
- 회의 주최자가 관련 태스크의 acceptance_criteria 를 모른 채 논의를 시작.
- 세션이 길어질수록 맥락이 쌓이지만, 다음 새 세션에서는 그 맥락이 날아감.

## 해결
`src/core/agent/prompt-builder.ts` 에 `buildContextBootstrapRules(agentRole)` 메소드를 추가하고, `buildSystemPrompt` 에서 **진실성 원칙 다음 우선순위**로 주입.

규칙 핵심:

1. **필수 로딩 순서** — kanban.json(본인 태스크) → sprints.json(현재 스프린트) → docs/decisions/*.md → docs/meetings/*.md(본인 참여) → docs/{role}/** → metrics.json
2. **로딩 완료 전 모든 산출 행동 금지** — 파일 수정·메시지 전송·태스크 이동 불가
3. **완료 신호 강제** — 정확히 `맥락 로딩 완료: N개 파일 확인 — 핵심 맥락: <한 문장 요약>` 을 출력해야 세션 로그에 검증 흔적이 남는다
4. **없는 파일은 `[없음]`** 으로 표시하고 스킵 — 실패 반복 금지
5. **5분 초과 시** 우선순위 1·2·3 만 완료하고 작업 진행

## 왜 "진실성 다음" 우선순위인가
- 진실성 규칙: "모르면 모른다고 말하라, 지어내지 말라" — 인식론적 preconditon
- 맥락 로딩 규칙: "모르는 걸 없애기 위해 파일을 먼저 읽어라" — 진실성의 실행 수단
- 둘은 짝이며, 맥락 로딩이 없는 진실성은 공허하고, 진실성 없는 맥락 로딩은 출처 왜곡으로 이어진다.

## 왜 Role 이름이 프롬프트에 하드코딩되나
역할명(`cto`, `po`, `ceo` 등)이 템플릿에 직접 치환되어 들어가야, 에이전트가:
- `kanban.json` 에서 `assignee === "<role>"` 로 필터링 가능
- `docs/<role>/` 같은 역할별 디렉토리를 확인 가능
- Glob 패턴(`docs/**/{<role>*,*<role>*}`)으로 역할별 문서 탐색 가능

추상 문구("본인 역할")만 써두면 에이전트가 구체 쿼리로 변환할 때 실수를 많이 내므로,
문자열 보간을 통해 **질의 가능한 형태**로 명시한다.

## 적용 범위
`PromptBuilder.buildSystemPrompt` 를 경유하는 **모든 세션**:
- `startAgent` — 정규 역할 에이전트 (CEO/CTO/PO/Designer/QA/Marketer)
- `startEphemeralAgent` — 자문 전문가 (역할명은 `consult-<uuid>` 식이라 Glob 결과 대부분 없음 → 자연스럽게 skip)
- `startMeetingSession` — 회의 주최자 (`buildSystemPrompt({ agent: initiator, ... })` 경유)

## 관련 테스트
- `tests/unit/core/prompt-builder.test.ts > 세션 시작 맥락 로딩 규칙이 역할명과 함께 주입된다`
  - 역할명 치환 확인, 섹션 순서(진실성 → 맥락 로딩 → 메트릭스) 검증

## 교훈
- "작업 시작 전 관련 문서를 확인하라" 같은 모호한 지시는 에이전트가 무시·축소한다. **어떤 파일을**, **어떤 순서로**, **무엇을 출력하면 끝인지** 까지 구체화해야 준수율이 오른다.
- 세션 간 맥락 단절은 멀티 에이전트 시스템의 근본 비용이다. 프롬프트를 거대하게 만들지 말고, "첫 행동으로 파일을 읽어라" 규칙 + 완료 신호 검증으로 외재화하는 편이 확장성이 좋다.
